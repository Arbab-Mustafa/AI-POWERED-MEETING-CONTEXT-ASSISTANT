STEP-BY-STEP COPY-PASTE INSTRUCTIONS
================================================================================

STEP 1: Create backend/.env file
---------
1. Navigate to: backend/ folder
2. Right-click → New → Text Document
3. Name it: .env (remove .txt)
4. Open the file
5. Copy everything from "EXAMPLE 1" above
6. Replace the placeholder values with YOUR actual values:
   - SECRET_KEY: Run Python command to generate
   - DATABASE_URL: Your PostgreSQL password
   - GOOGLE_CLIENT_ID: From Google Cloud Console
   - GOOGLE_CLIENT_SECRET: From Google Cloud Console
   - GMAIL_EMAIL: Your Gmail address
   - GMAIL_APP_PASSWORD: From Google app passwords
   - TELEGRAM_BOT_TOKEN: From BotFather

STEP 2: Create frontend/.env.local file
---------
1. Navigate to: frontend/ folder
2. Right-click → New → Text Document
3. Name it: .env.local
4. Open the file
5. Copy everything from "EXAMPLE 2" above
6. Replace placeholder values with YOUR actual values:
   - NEXT_PUBLIC_API_URL: http://localhost:8000

STEP 3: Verify files are in correct locations
---------
After creation, files should be at:
- c:\Users\hp\Desktop\Managment_agent\backend\.env
- c:\Users\hp\Desktop\Managment_agent\frontend\.env.local

NOT in subdirectories!

================================================================================
COMMON MISTAKES TO AVOID
================================================================================

❌ WRONG:
DATABASE_URL = postgresql://postgres:Password@localhost:5432/contextmeet
(Space around = sign)

✅ CORRECT:
DATABASE_URL=postgresql://postgres:Password@localhost:5432/contextmeet
(No spaces around =)

---

❌ WRONG:
GOOGLE_CLIENT_SECRET="GOCSPX-abc123def456"
(Quotes around value)

✅ CORRECT:
GOOGLE_CLIENT_SECRET=GOCSPX-abc123def456
(No quotes)

---

❌ WRONG:
SECRET_KEY=A-8K_mZ9pQwXyZ1jH4tL5vN6rB2cD3fG4hJ7kL9mN1pR3sT5uV7wX9yZ # My secret
(Comments with #)

✅ CORRECT:
SECRET_KEY=A-8K_mZ9pQwXyZ1jH4tL5vN6rB2cD3fG4hJ7kL9mN1pR3sT5uV7wX9yZ
(No comments after values)

---

❌ WRONG:
ALLOWED_ORIGINS=http://localhost:3000, http://localhost:3001
(Spaces after comma)

✅ CORRECT:
ALLOWED_ORIGINS=http://localhost:3000,http://localhost:3001
(No spaces)

---

❌ WRONG:
TELEGRAM_BOT_TOKEN=6123456789:ABCDefGh IjKlMnOp
(Spaces in middle of token)

✅ CORRECT:
TELEGRAM_BOT_TOKEN=6123456789:ABCDefGhIjKlMnOp
(No spaces)

================================================================================
SECURITY BEST PRACTICES
================================================================================

1. NEVER commit .env files to GitHub
   - They're already in .gitignore ✅
   - Double-check before pushing

2. NEVER share SECRET_KEY, GOOGLE_CLIENT_SECRET, GMAIL_APP_PASSWORD
   - These are confidential
   - If leaked, regenerate immediately

3. NEVER use same password for Gmail and other services
   - Gmail app password is for THIS app only
   - Your actual Gmail password remains separate

4. NEVER hardcode credentials in code
   - Always read from .env files
   - We already do this ✅

5. ROTATE tokens periodically
   - Google: Regenerate OAuth credentials every 6 months
   - Telegram: Can regenerate token via BotFather
   - Mistral: Not applicable for local Ollama

6. Monitor API usage
   - Google gives 1M requests/day (free tier)
   - Telegram has no hard limits
   - Mistral local = no cost

================================================================================
WHAT HAPPENS WHEN VARIABLES ARE MISSING
================================================================================

Missing Variable              | Error You'll See
------------------------------|----------------------------------------
SECRET_KEY                    | KeyError: 'SECRET_KEY' not found
DATABASE_URL                  | Connection error to PostgreSQL
GOOGLE_CLIENT_ID              | OAuth login fails with 400 error
TELEGRAM_BOT_TOKEN            | Telegram messages won't send
MISTRAL_BASE_URL              | Cannot connect to Ollama (AI features fail)
GMAIL_APP_PASSWORD            | Email sending fails with 401 auth error
NEXT_PUBLIC_API_URL           | Frontend can't reach backend (404)

If you see any of these errors, review the corresponding variable in your .env

================================================================================
NEXT STEPS AFTER SETTING UP .ENV
================================================================================

1. Install backend dependencies:
   cd backend
   pip install -r requirements.txt

2. Install frontend dependencies:
   cd frontend
   npm install

3. Start Ollama (if not running):
   ollama serve

4. Start postgres (if local):
   Ensure PostgreSQL service is running

5. Start Backend:
   cd backend
   python -m app.main

6. Start Frontend (in new terminal):
   cd frontend
   npm run dev

7. Access application:
   Frontend: http://localhost:3000
   Backend API: http://localhost:8000
   API Docs: http://localhost:8000/docs
